How Kong Rate Limiting Works

                    

                        

                            
1

                            

                                Request arrives at Kong
                                
Kong intercepts every request before it reaches your API service.

                            

                        

                        

                            
2

                            

                                Rate limit counter checked in Redis
                                
Kong queries Redis to check how many requests this IP/consumer has made in the current window.

                            

                        

                        

                            
3

                            

                                Under limit → forward to API
                                
Kong adds X-RateLimit-Remaining-minute header and proxies the request.

                            

                        

                        

                            
4

                            

                                Over limit → 429 Too Many Requests
                                
Kong returns 429 immediately — the API service never even sees the request.
