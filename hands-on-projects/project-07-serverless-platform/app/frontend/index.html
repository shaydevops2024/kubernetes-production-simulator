What is FaaS?

1
You deploy a function, not a server

Instead of managing a running process 24/7, you upload code. The platform handles everything else: containers, scaling, networking.

2
An event triggers execution

HTTP request, Kafka message, Cron schedule — any event can trigger your function. No event = no running container = no cost.

3
Platform scales from zero automatically

First request might be slow (cold start). Subsequent requests hit a warm container. Under high load, Kubernetes scales out replicas automatically.

4
Your DevOps job: deploy & configure the platform

Install OpenFaaS or Knative, configure ingress, set up autoscaling policies, manage the function lifecycle via CI/CD. The app is pre-built — the infrastructure is yours to build.

Scaling Concepts

                    

                        

                            
❄

                            
Cold Start

                            
First request to a scaled-to-zero function. Kubernetes must schedule a pod, pull the image, start the container. Adds 1–10s latency. Solution: keep-alive probes, pre-warming, min replicas.

                        

                        

                            
☀

                            
Warm Container

                            
Subsequent requests hit an already-running container. Near-instant response. Kubernetes keeps the pod alive for a configurable idle timeout before scaling to zero.

                        

                        

                            
📈

                            
Scale-Out (HPA)

                            
Under high load, HPA (Horizontal Pod Autoscaler) adds replicas based on CPU/memory/custom metrics. With KEDA, you can scale on HTTP request queue depth too.

                        

                        

                            
🔴

                            
Scale-to-Zero

                            
Knative and OpenFaaS can scale pods to zero when idle. Zero cost, zero resource usage — but at the cost of cold starts. Configurable per function.

                        

                    

                

            


            
            

                

                    
Trigger Types

                    
Functions can be invoked by different event sources — this tab explains each one

                


                

                    

                        

                            🌐
                            

                                
HTTP Trigger

                                Active in this demo
                            

                        

                        

                            
The most common trigger. An HTTP request (GET, POST, etc.) to a URL invokes the function. OpenFaaS and Knative both expose an HTTP endpoint per function.

                            

                                
Example (curl)

                                
curl -X POST http://faashub/function/hello-world \
  -H "Content-Type: application/json" \
  -d '{"name": "DevOps"}'

                            

                            

                                
What you configure in K8s

                                
kind: Function   # OpenFaaS CRD
metadata:
  name: hello-world
spec:
  image: your-registry/hello-world:v1
  labels:
    com.openfaas.scale.zero: "true"

                            

                        

                    


                    

                        

                            ⏱
                            

                                
Cron Trigger

                                Scheduled
                            

                        

                        

                            
Schedule a function to run on a cron expression. Perfect for periodic tasks: reports, data sync, cache warming, health checks. In Kubernetes, this is a CronJob or a KEDA ScaledJob.

                            

                                
Example (Kubernetes CronJob)

                                
apiVersion: batch/v1
kind: CronJob
metadata:
  name: weather-report-cron
spec:
  schedule: "0 * * * *"   # every hour
  jobTemplate:
    spec:
      template:
        spec:
          containers:
          - name: invoker
            image: curlimages/curl
            args:
            - curl -X POST http://faashub/function/weather-report
              -d '{"city": "Tel Aviv"}'

                            

                        

                    


                    

                        

                            📤
                            

                                
Kafka Trigger

                                Event-Driven
                            

                        

                        

                            
A message on a Kafka topic triggers function invocation. KEDA watches the consumer group lag and scales replicas accordingly. Zero messages = zero pods. Burst of messages = instant scale-out.

                            

                                
KEDA ScaledObject for Kafka

                                
apiVersion: keda.sh/v1alpha1
kind: ScaledObject
metadata:
  name: text-processor-kafka
spec:
  scaleTargetRef:
    name: text-processor
  triggers:
  - type: kafka
    metadata:
      topic: text-events
      bootstrapServers: kafka:9092
      consumerGroup: text-processor
      lagThreshold: "5"
