# Pod Disruption Budget (PDB)

## Objective
Ensure high availability during voluntary disruptions using Pod Disruption Budgets.

## What You'll Learn
- Create Pod Disruption Budgets
- Set minAvailable and maxUnavailable
- Test PDB enforcement
- Handle node maintenance safely
- Balance availability and updates

## Duration
20 minutes

## Success Criteria
- PDB created and active
- PDB prevents disruption beyond limits
- Application remains available during drain
- All pods recover successfully
